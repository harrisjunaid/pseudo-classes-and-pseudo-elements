pseudo-classes for document structure

:only-child & :only-of-type

:only-child
Selects elements with no siblings. Examps are code element within this description title and the strong element within this dd tag of dt.
:only-of-type
Selects distinct type unique elements among siblings. For instance both of this article's children h2 and dl are of separate type.

:first-child,     :first-of-type & :last-child,    :last-of-type

:first-child
select the first child among siblings.
:first-of-type
select the first child of defined type among siblings.
:last-child
selects the last child among siblings.
:last-child
selects the last child among siblings.

:nth-child,         :nth-of-type&:nth-last-child,  :nth-last-of-type

:nth-child
nth child from the beginning
:nth-of-type
nth distint type child from the beginning
:nth-last-child
nth child counting from end or the last child
:nth-last-of-type
nth distint type child counting from end or the last child

:root & :empty

:root
setting up the root of the document.
  • Pseudo-classes do have a higher level of specificity than the HTML element does in CSS, and so anything put in root would win over anything put in the body tag or the HTML tag.
  • Also allow you to access the shadow DOM, if that winds up being something you want to do with Javascript.
  • becoming really commonly popular with CSS variables, also called CSS custom properties
:empty
tag that exists, but it has no content and no whitespace in it. Mostly used for altering with programming language
